# Issue tracker: local markdown

Issues and PRDs for this repo live as markdown files under `.scratch/issues/`.

## Conventions

- **Create an issue**: add a new file under `.scratch/issues/` with a numbered slug name, such as `01-bootstrap-deployable-monorepo.md`.
- **Read an issue**: open the matching markdown file under `.scratch/issues/`.
- **List issues**: inspect the files in `.scratch/issues/`, usually in numeric order.
- **Comment on an issue**: add notes directly in the issue file.
- **Apply / remove labels**: encode labels in the issue frontmatter or body if the file format uses them.
- **Close**: mark the issue as done in the file contents or move it to an archive convention if the repo adopts one.

## When a skill says "publish to the issue tracker"

Create a markdown issue file under `.scratch/issues/`.

## When a skill says "fetch the relevant ticket"

Open the matching file under `.scratch/issues/`.
